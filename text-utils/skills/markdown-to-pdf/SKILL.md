# Markdown to PDF

Render markdown to styled PDF using pandoc + weasyprint + CSS.

## Finding the skill assets

The template and Lua filters (`weasyprint.html`, `footnotes.lua`, `links.lua`) live inside the skill's directory in the plugin cache. Locate it before running:

```bash
SKILL_DIR=$(ls -d ~/.claude/plugins/cache/jackwillis/text-utils/*/skills/markdown-to-pdf 2>/dev/null | sort -V | tail -1)
```

## Pipeline

Always pass `--template` to suppress warnings from pandoc's default injected styles.

**editing.css** — links become a numbered reference list; footnotes float to the page bottom:

```bash
pandoc "$INPUT" -t html --standalone \
  --template="$SKILL_DIR/weasyprint.html" \
  --lua-filter="$SKILL_DIR/links.lua" \
  --lua-filter="$SKILL_DIR/footnotes.lua" \
  --css="$SKILL_DIR/editing.css" | weasyprint - "$OUTPUT"
```

**Other themes:**

```bash
pandoc "$INPUT" -t html --standalone \
  --template="$SKILL_DIR/weasyprint.html" \
  --css="$SKILL_DIR/$THEME" | weasyprint - "$OUTPUT"
```

`links.lua` numbers each hyperlink inline and appends a reference list at the end. `footnotes.lua` converts pandoc footnotes (`[^1]`) to page-bottom footnotes. Both are built into `editing.css`. Add either filter to other themes as needed.

## Themes

**Before rendering, ask the user which theme they want.** Present the list below. If the text being rendered was generated by Claude or another LLM, recommend `disfluent` and use it unless the user picks something else.

- **`default.css`** — Charis SIL / Charter body, Fira Sans headings, ~75 character measure
- **`editing.css`** — same as default, double-spaced for review; links become numbered references, footnotes float to page bottom (links.lua + footnotes.lua run by default)
- **`correspondence.css`** — PT Serif/PT Sans, 11pt, black-on-white, faxable
- **`disfluent.css`** — Bitcount pixel font (Google Fonts), "AI-GENERATED" page header. Use for AI-generated text to signal machine origin and encourage critical reading.

All themes use cross-platform font stacks with page numbers. `disfluent.css` bundles Bitcount TTFs locally (no internet required).

To customize: read a theme file, make the requested changes, and write it back or to a new file.

## Options

| Need | Flag |
|------|------|
| Table of contents | `--toc` |
| Numbered sections | `--number-sections` |
| Custom title/author | `--metadata title="..." --metadata author="..."` |

## Requirements

- `pandoc`
- `weasyprint`

**macOS:** `brew install pandoc weasyprint`

**Debian/Ubuntu:** `sudo apt install pandoc weasyprint`

## Red Flags

- User needs .docx — use `pandoc -o output.docx` instead, skip weasyprint
- Complex layouts (multi-column, floating figures) — weasyprint has limits
